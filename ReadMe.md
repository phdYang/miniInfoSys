## 2.编程语言的不同

###  语法层次
* 基本上各个语言都几乎一样，都是基本的变量定义赋值语句加上基本的循环控制语句

###  语言思想层次
* 不同的语言有自己特有的思想，而现在的主流是面向对象的思想，这个思想需要自己好好琢磨，从标准答案来说， 面向对象的要素是封装继承和多态(我们这个系统其实就是诠释了封装的含义，而且还是封装的基本含义)。同时，C++/Java/Python也同样都是面向对象，所以这个思想还是挺好的，但同时，我个人认为，思想不免有些“哲学”“玄学”在里面。

###  工具层次
* 每个语言都有自己的一套最合适的工具，或者我们称之为IDE(集成开发环境)，原则上大牛喜欢用自己的编辑器来配置自己的开发环境，比如说sublime text/VScode/vim/emacs/NotePad++ 但我依然建议我们掌握IDE
* 对于C/C++来说，或者对于微软的开发产品（.Net/C#/SQL Server/basic）,我们说visual studio的学习难度不亚于C++本身，这是一款号称宇宙最强的IDE,还是很复杂的
* 对于Python来说，自带的IDLE除了方便没有其他优势，甚至还不如编辑器，所以不算是IDE;我们说PyCharm是使用最广的Python的IDE，因为Python也是一款可以做开发的轻量级的优秀的开发语言，比如Flask/Django;当然还有具有强大的包管理机制的Anaconda,如果是做数据分析，或者跑一些实验编程TensorFlow，我还是很建议这个软件，功能强大，自带很多第三方库免去一个个下载的痛苦
* 对于Java来说，Eclipse/MyEclipse是一直以来最为老牌使用最广泛的一款IDE,大部分选择使用开源的Eclipse，因为开源，我们可以使用插件来自定义IDE;还有IDEA(IntelliJ IDEA)这个也是广泛使用的IDE,并且功能更为强大，非常适合用来开发。

##   4.编程和开发(这两个词其实找到不好，我并没有求证)
   
> 编程必然是开发的基础，但二者并不是显然的包含与被包含的关系

1. 开发涉及到**软件工程**/**软件项目管理**/**软件设计模式**的一些思想，和工业界约定俗成的规范（比如命名规范(蜂窝式拼写法)/）开发多使用一些工业界大都使用的稳定的具有多场景扩展性的框架，比如SpringMVC，相比于性能和高效，更看重代码的规范和可维护性(**初级工程师**)其次，开发更重视一些高并发多线程的一些使用场景(**高级工程师**)和一些系统开发的架构(__系统架构师__)
        
2. 编程更多的着眼于**算法**，是一个函数的具体的设计(换句话说，开发更多的是使用算法，编程更对的是设计算法)。一个好的开发产品背后必然有好的算法支持。比如说，微信的听歌识曲功能，核心是语音识别的算法(这是一篇篇论文和数学公式以及很多的实验推导出来的算法，经过了很多的演变)，开发不需要知道这些，包括论文都不需要看，在他们眼里，都是一样的一行代码调用和int a = 1;并没有太多的不同
       
3. 编程能力的提升
* 首先知道编程是分三部分，IPO.也就是输入/处理/输出。任何一个程序，我们永远考虑的是输入是什么；输出是什么；我们怎么设计函数来保证y = f(x);
* 其次就是练习，计算机本质依旧是操作大于理论的课程，没有练习就像没有下过水的游泳学习者一样罢了，可以了解ACM(计算机界的奥利匹克程序大赛)的一些东西，练习刷题的东西有牛客网;LeetCode;各大高校的OJ（清华oj/北大oj/浙大oj/杭电oj）
